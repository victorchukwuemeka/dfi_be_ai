# Assignment: Collaborative Document with Version History

## Task
Create a document through iterative human-AI collaboration, tracked in Git.

## Requirements
1. Choose a document type (proposal, report, guide, article)
2. Iterate through at least 4 versions using this cycle:
   - AI draft → commit
   - AI self-review → commit
   - Human edits → commit
   - Final polish → commit
3. Use at least one branch for an alternative approach
4. Merge the branch or document why you didn't

## Deliverable
Submit via one of:
- GitHub/GitLab repo link
- Zipped repo with .git folder

Include a brief write-up (200 words) answering:
- What did the AI self-review catch that you wouldn't have?
- What was the most valuable human edit you made?
- How did branching help (or not)?

## Grading Criteria
- 4+ versions committed
- AI self-review used at least once
- Branching used
- Commit messages are descriptive
- Reflection shows understanding of the collaboration loop
