# Week 12: Capstone Preparation

## Capstone Overview
Your capstone is an end-to-end AI-assisted workflow that solves a real problem. The goal is to demonstrate everything you've learned: scoping, prompting, iteration, verification, version control, and documentation.

## Choosing Your Capstone

### Good Capstone Criteria
- Solves a real problem you face (or someone you know faces)
- Has measurable before/after impact
- Involves at least 3 distinct AI interactions (not one prompt)
- Can be documented clearly

### Examples
- **Content workflow**: Topic → research → outline → draft → refine → publish (track time saved)
- **Decision support**: Problem → options → analysis → recommendation → action plan
- **Research synthesis**: Sources → extract → patterns → brief → verification
- **Communication toolkit**: Input → multiple formats (memo, email, brief) → review → final
- **Process automation**: SOP → prompt chain → test → refine → documentation

## Capstone Requirements (Preview)

### Deliverables
1. **Problem statement** — what are you solving and why does it matter?
2. **Before state** — how was this done before? (time, quality, effort)
3. **Workflow** — step-by-step with all prompts used
4. **Evidence** — outputs at each stage
5. **Refinement** — what changed between version 1 and final?
6. **After state** — what improved? (quantify if possible)
7. **Version history** — Git log showing the full evolution
8. **Reflection** — what worked, what didn't, what would you do differently?

## This Week: Proposal
Submit a one-page proposal for your capstone including:
- Problem description
- Proposed workflow (high-level steps)
- Expected impact (before/after)
- Success criteria (how will you know it worked?)

Use AI to help draft and refine the proposal. Apply the full prompting framework.
