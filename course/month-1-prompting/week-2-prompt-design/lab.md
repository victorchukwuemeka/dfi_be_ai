# Lab: Build a Structured Prompt

## Objective
Practice writing prompts using the Role-Task-Constraints-Examples-Output framework.

## Instructions

### Part 1: Identify Components
Look at this prompt and identify each component:

"You are a project manager writing to a cross-functional team. Summarize the following status update. Keep it under 5 bullet points. Use a neutral, factual tone. End with one question for the team.

Status: [text]"

### Part 2: Rewrite a Weak Prompt
Original: "Tell me about marketing."

Rewrite it using all five components.

### Part 3: Prompt A/B Test
Write two versions of a prompt for the same task (e.g., "explain a concept from your field"):

- Version A: Role + task only
- Version B: Role + task + constraints + examples + output schema

Compare outputs. Which is more usable? Why?

## Deliverables
- Your rewritten prompt from Part 2
- Both outputs from Part 3 with your comparison notes
